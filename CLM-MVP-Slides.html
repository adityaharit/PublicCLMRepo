The Problem

Without a central system, basic contract questions cannot be answered — creating audit risk and costly missed renewals.

"Where is the signed MSA with Vendor X?"

"Who approved this contract, and when?"

"Which agreements are expiring in the next 60 days?"

"Has anyone changed this document since it was approved?"

CLM Promise

Contracts live in email threads, shared drives, and personal inboxes. CLM makes these questions trivially answerable — governance visible, auditable, and enforceable.

2

Lifecycle & Scope

The MVP delivers four of five contract stages natively, with integrations to three enterprise systems planned in the future.

Intake & Request

Submit requests; metadata pre-filled from commercial estimating

Generation & Negotiation

Build from approved templates; redlining stays in Word

Governance & Approvals

Multi-stage approval chains with SLA tracking and immutable audit trail

Execution & Obligation

Execute agreed contract; auto-trigger PO and track obligations

Repository & Reporting

Search contracts, manage renewals, and export portfolio data

Deferred from MVP

E-signature integration deferred. In-document redlining stays in Microsoft Word for this release. All other stages are fully functional at launch.

Secure User Login

User Story

As a Platform User, I want to log in securely via SSO or email and password so that I can reach my contracts without creating access risk.

Acceptance Criteria

  • Login via SSO or email/password; session expires after configurable inactivity period
  • Five consecutive failed login attempts locks the account; email notification sent to user
  • Password reset via time-limited email link
  • Every login and logout recorded in audit log with timestamp and IP address

Edge Case

SSO outage: System falls back to email/password if SSO provider times out

Role-Based Permission Enforcement

User Story

As an Admin, I want the system to enforce strict role-based access limits so that no one accidentally edits, approves, or shares beyond their authority.

Acceptance Criteria

  • Viewer: read-only access to explicitly granted contracts only
  • Procurement: upload contracts, edit metadata
  • Legal: Procurement permissions + initiate workflows + manage templates
  • Admin: full access including user/workflow management
  • Forbidden actions are hidden in UI AND blocked at the server API level

Edge Case

Role downgrade mid-session: Session automatically refreshes to enforce new permissions immediately

Contract-Level Permission Overrides

User Story

As a Legal User or Owner, I want to grant temporary View or Edit access to a specific colleague on a single contract so that I can collaborate precisely and revoke access when done.

Acceptance Criteria

  • Owners with Legal role or above can add specific users as View or Edit collaborators on a single contract
  • Grants and revocations take effect immediately with no page reload delay
  • All grant and revocation actions logged in audit trail (who granted, to whom, when)

Edge Case

Deactivated collaborator: If a collaborator's account is deactivated, their contract-specific grants are gracefully ignored

User Account Provisioning & Off-boarding

User Story

As an Admin, I want to provision or deactivate user accounts so that system access stays synchronised with employment status.

Acceptance Criteria

  • Admin creates accounts with name, email, and role; 48-hour invite link sent automatically
  • Deactivated users cannot log in; their names persist on all historical records
  • All account lifecycle events (create, role change, deactivate) logged

Edge Case

Expired invite: System displays clear message directing user to request a new link from Admin

User Story

As a Procurement User or above, I want to upload a final agreement to the platform so that it is accessible to colleagues and securely stored.

Acceptance Criteria

  • Every metadata edit logs the field name, old value, new value, user, and timestamp

  • All columns sortable; sort state persists within session
  • Results scoped to role-permitted or explicitly granted contracts only

Parent-Child Contract Linking

As a Procurement User or above, I want to link amendments or renewals to a parent contract so that the full family of agreements is visible in one place.

  • Parent contract shows a "Related Agreements" list; child shows a back-link to its parent

  • Page loads in under 3 seconds for documents up to 25MB

Corrupted preview: If PDF viewer fails to render, a fallback "Download Document" button remains functional at all times

The Six-State Lifecycle

User Story

As the System, I want to enforce strict lifecycle phases (Draft → In Review → Approved → Executed → Expired) so that no contract bypasses required stages.

Acceptance Criteria

  • Linear path strictly enforced; invalid transitions blocked and hidden from UI
  • Terminated/Cancelled reachable from any active state (Draft, In Review, Approved, Executed)
  • All state changes logged with: from-state, to-state, user, timestamp

Edge Case

Concurrent transition: If two authorised users transition simultaneously, first is processed; second receives "State already updated" message

Terminated / Cancelled State

User Story

As a Legal Ops User or Admin, I want to transition a contract to Terminated/Cancelled with a recorded reason so that I can remove it from active workflows without losing its history.

Acceptance Criteria

  • Reachable from Draft, In Review, Approved, or Executed states only
  • Mandatory reason note required (minimum 20 characters)
  • Record locks immediately on transition; no further edits possible

Edge Case

Already expired: System prevents Terminated/Cancelled transitions on contracts already in the Expired terminal state

Automatic Expiry on End Date

User Story

As the System, I want to transition contracts to Expired automatically when their end date passes so that the portfolio is always accurate without manual tracking.

Acceptance Criteria

  • Daily CRON job checks all contracts in Executed state
  • Contracts with End Date ≤ today are transitioned to Expired automatically
  • Contract owner notified via in-app notification on automatic expiry

Edge Case

Job failure: If daily job fails, it retries automatically every hour until successful; failure is logged in Admin audit trail

Lock Approved and Closed Contracts

User Story

As an Auditor/System, I want Approved, Executed, Terminated, and Expired contracts permanently locked against edits so that agreed terms are preserved intact.

Acceptance Criteria

  • All metadata fields and document edit controls locked on reaching these states
  • Admin can unlock a contract to Draft state but must enter a mandatory justification
  • Unlock justification is permanently and immutably logged

Edge Case

Admin unlock alerts: Unlocking triggers automatic email to Legal Ops and the contract owner to prevent silent administrative overrides

Stakeholder Notifications on State Change

User Story

As a Contract Owner, I want automatic notifications when my contract changes state so that I can keep workflows moving efficiently.

Acceptance Criteria

  • In-app and email notification sent within 60 seconds of any state change
  • Notification includes: previous state, new state, actor who made the change, and a direct link to the contract

Edge Case

Email bounce: If email provider fails, in-app notification is still reliably delivered; email failure logged but does not block the state transition

Immutable Contract Audit Log

User Story

As a Compliance Officer, I want an uneditable, permanent log of every contract action so that I can demonstrate compliance during audits.

Acceptance Criteria

  • Logs: create, upload, edit (field-level), state transition, approval decision, collaborator grant/revoke, view
  • Each entry in plain language format: who, what, when
  • Read-only for all roles including Admin; filterable by date range and action type; exportable to CSV

Edge Case

Massive export: If a global log export exceeds standard row limits, the system paginates CSV or delivers it asynchronously via email link

  • Extracted variable list is displayed to the author for verification before saving
  • Malformed placeholders (e.g. unclosed brackets) are flagged as warnings

Generate a Contract from a Template

User Story

As a Procurement User, I want to fill in a web form to generate a contract so that I produce accurate, compliant documents in minutes without Legal drafting from scratch.

Acceptance Criteria

  • On submit: substitutes all variables, compiles PDF and DOCX, creates a Draft contract record linked to the template version used

Edge Case

Generation timeout: If document compilation takes >30 seconds, UI shows a friendly loading state; the process does not crash or lose user input

Builder Validation & Error Handling

User Story

As a Business User, I want instant validation on my form inputs so that I can fix errors before generation fails.

Acceptance Criteria

  • Real-time inline validation on dates (valid format, chronology), currencies (numeric), and required fields
  • Failed generation attempt preserves all form entries so work is not lost
  • Clear error message identifies which field caused the failure

Edge Case

Extreme inputs: Numeric currency fields enforce maximum limits to prevent integer overflow errors in the database

Configure Sequential & Parallel Workflows

User Story

As an Admin, I want to build approval routing templates so that specific contract types route correctly and automatically without manual intervention.

Acceptance Criteria

  • Build named workflow templates with ordered stages; each stage can be sequential or parallel
  • Workflow templates are associated with specific contract types
  • Active workflow templates cannot be edited; a new version must be created instead

Edge Case

Empty stage: UI prevents saving a workflow template if a stage exists but no approvers are assigned to it

Approval Stage SLA Configuration

User Story

As an Admin, I want to set SLAs on approval stages so that the system tracks bottlenecks automatically rather than relying on manual follow-up.

Acceptance Criteria

  • Configurable SLA duration (e.g. 48 hours) set per stage within a workflow template
  • Visual amber warning applied to any stage that has exceeded its SLA
  • SLA breach triggers notification to the contract owner

Edge Case

Zero-day SLA: Form validation requires SLA to be greater than 0

Submit a Contract for Approval

User Story

As a Legal User, I want to trigger the approval chain from the contract page so that stakeholders are formally engaged with a single action.

Acceptance Criteria

  • "Submit for Approval" button visible to Legal and Admin roles on contracts in Draft state
  • System verifies a workflow template exists for the contract's type before allowing submission
  • On submit: contract transitions to In Review; first-stage approvers notified immediately

Edge Case

No workflow configured: Submission blocked with message: "No approval workflow configured for this Contract Type. Contact Admin."

Approve or Reject with Mandatory Comment

User Story

As an Approver, I want to record my decision in the platform so that my sign-off or objection is officially and permanently captured.

Acceptance Criteria

  • Approve action records decision immediately with timestamp
  • Reject action requires a mandatory text comment before it can be submitted
  • Both decisions written immediately to the audit log

Edge Case

Role change mid-flight: If an approver loses their role while a contract is pending, the stage is flagged "Unassigned" for Admin reassignment

Automatic Chain Advancement

User Story

As the System, I want to automatically advance contracts to the next stage upon approval so that the workflow never stalls waiting for manual progression.

Acceptance Criteria

  • Sequential: completion of one stage immediately triggers the next
  • Parallel: all approvers in the stage must approve before the chain advances; a single rejection halts the entire chain
  • Final stage approval automatically transitions the contract to Approved state

Edge Case

Simultaneous parallel rejection: If two parallel approvers reject at the exact same microsecond, both rejection comments are preserved in the audit log

Rejection & Resubmission Flow

User Story

As a Contract Owner, I want to be notified of rejections and easily resubmit so that I can resolve issues without losing the historical approval chain.

Acceptance Criteria

  • Owner notified within 60 seconds of rejection with the full rejection comment and rejecting approver's name
  • Contract reverts to Draft state on rejection
  • All prior approval records (accepted and rejected stages) are retained in the audit log
  • Resubmitting restarts the approval chain from Stage 1

Edge Case

Resubmission without document change: UI allows resubmission even if the document was not changed, as the underlying blocker may have been resolved off-system

Approval SLA Breach Creates RAID Issue

User Story

As the System, I want to post an Issue to Palantir RAID when an approval SLA is breached so that governance blockages escalate to leadership automatically.

Acceptance Criteria

  • SLA breach exceeding 48 hours automatically creates a RAID Issue in Palantir
  • Issue includes: contract name, overdue approver, SLA due date, and direct link to the contract
  • Exactly one Issue created per breach event — no duplicate Issues on repeat checks

Edge Case

API timeout: If Palantir API fails, system retries every 15 minutes; each failure logged as "System Integration Failure" in Admin audit trail

Contract Execution Creates RAID Decision

User Story

As the System, I want to record major contract executions as a RAID Decision so that the wider team receives an official signal to proceed.

Acceptance Criteria

  • Contracts transitioning to Executed with total value above a configurable threshold create a RAID Decision in Palantir
  • Decision includes: vendor name, contract value, execution date, and link

Edge Case

Missing value: Contracts with zero total value (e.g. zero-dollar NDAs) bypass this logic unless overridden by Admin configuration

Approaching Expiry Without Renewal Creates RAID Risk

User Story

As the System, I want to raise a RAID Risk for contracts 30 days from expiry without a renewal so that unmanaged portfolio exposure is highlighted.

Acceptance Criteria

  • Triggers when a contract reaches 30 days before its End Date and has no linked child "Renewal" record
  • Linking a Renewal record to the contract subsequently closes the RAID Risk automatically

Edge Case

Terminated contracts: Contracts manually moved to Terminated/Cancelled bypass the expiry risk logic entirely

Admin Contract Unlock Creates RAID Risk

User Story

As the System, I want to create a RAID Risk when an Admin unlocks an approved contract so that post-approval modifications are visible to leadership.

Acceptance Criteria

  • Unlocking an Approved or Executed contract back to Draft state creates a RAID Risk in Palantir
  • Risk includes: Admin name, justification text entered at unlock, date

Edge Case

Immediate relock: Even if the Admin relocks the contract seconds later, the RAID Risk persists for audit review

RAID Resolution Notifies CLM Contract Owner

User Story

As a Contract Owner, I want to be notified when my linked RAID item is resolved so that I can unblock my workflow without manually monitoring Palantir.

Acceptance Criteria

  • A RAID Issue or Risk being resolved in Palantir triggers both an in-app and email notification to the CLM contract owner within 60 seconds
  • Notification includes: resolution note text and a direct link to the relevant contract

Edge Case

Reopened RAID item: If a resolved item is subsequently reopened in Palantir, the CLM contract owner receives a corresponding "RAID Item Reopened" alert
